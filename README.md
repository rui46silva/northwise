# Northwise

> **Direction over noise.**
> A web development studio in Northern Portugal — strategic websites, landing pages and custom web platforms.

SEO-first marketing landing page built with **Next.js (App Router)**, **TypeScript** and **Tailwind CSS**. Includes a lead-generation contact form wired to **Resend** (email) and **HubSpot** (CRM), plus **Google Analytics 4**.

## Tech stack

- [Next.js 14](https://nextjs.org/) — App Router, server-side rendering for SEO
- [Tailwind CSS](https://tailwindcss.com/)
- [Zod](https://zod.dev/) — shared form validation (client + server)
- [Resend](https://resend.com/) — transactional email for lead notifications
- [HubSpot CRM](https://developers.hubspot.com/) — lead capture
- [Google Analytics 4](https://analytics.google.com/)

## Getting started

```bash
npm install
cp .env.example .env.local   # then fill in the values
npm run dev                  # http://localhost:3000
```

The site runs fine with no environment variables set — analytics and the
email/CRM integrations simply stay inactive until you configure them.

## Environment variables

| Variable | Required | Purpose |
| --- | --- | --- |
| `NEXT_PUBLIC_SITE_URL` | recommended | Canonical URL for SEO metadata, sitemap and Open Graph. |
| `NEXT_PUBLIC_GA_ID` | optional | Google Analytics 4 Measurement ID (`G-XXXXXXXXXX`). Analytics is off when empty. |
| `RESEND_API_KEY` | for the form | API key from Resend. Without it, the form returns a friendly error. |
| `CONTACT_FROM_EMAIL` | for the form | Verified sender address in Resend. |
| `CONTACT_TO_EMAIL` | for the form | Inbox that receives new lead notifications. |
| `HUBSPOT_ACCESS_TOKEN` | optional | Private App token (`crm.objects.contacts.write`). CRM sync is skipped when empty. |

### Setting up the integrations

**Resend (email)**
1. Create an account at [resend.com](https://resend.com) and verify your sending domain.
2. Create an API key → set `RESEND_API_KEY`.
3. Set `CONTACT_FROM_EMAIL` to an address on the verified domain and `CONTACT_TO_EMAIL` to where you want leads delivered.

**HubSpot (CRM)**
1. In HubSpot, go to *Settings → Integrations → Private Apps* and create an app.
2. Grant the `crm.objects.contacts.write` scope.
3. Copy the access token → set `HUBSPOT_ACCESS_TOKEN`. New leads appear as contacts (lifecycle stage: *lead*).

**Google Analytics 4**
1. Create a GA4 property and copy the Measurement ID.
2. Set `NEXT_PUBLIC_GA_ID`. IP anonymization is enabled by default.

## How the lead form works

1. The form (`src/components/Contact.tsx`) posts JSON to `POST /api/contact`.
2. The route (`src/app/api/contact/route.ts`) validates input with the shared Zod schema (`src/lib/lead.ts`), including a hidden honeypot field for spam.
3. It sends an email via Resend (the critical path), then best-effort syncs the contact to HubSpot.

## SEO features

- Full metadata, Open Graph and Twitter cards (`src/app/layout.tsx`)
- Dynamically generated Open Graph image (`src/app/opengraph-image.tsx`)
- JSON-LD structured data (`ProfessionalService`)
- `sitemap.xml` and `robots.txt` generated by Next.js
- Semantic HTML, single `h1`, descriptive headings, skip-to-content link
- Server-rendered content for crawlers

## Project structure

```
  app/
    layout.tsx              # metadata, fonts, JSON-LD, analytics
    page.tsx                # landing page composition
    globals.css             # theme + brand utilities
    opengraph-image.tsx     # dynamic OG image
    sitemap.ts / robots.ts  # SEO routes
    api/contact/route.ts    # lead handler (Resend + HubSpot)
  components/               # Nav, Hero, Solutions, Process, Care, Thinking, Contact, Footer…
  lib/
    site.ts                 # central site config / content
    lead.ts                 # shared form schema
```
